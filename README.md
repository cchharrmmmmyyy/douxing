# douxing

网络应用开发课程项目：体育运动明星主题展示。

## 项目结构

```text
douxing/
├─ index.html
├─ css/
│  └─ main.css
├─ js/
│  └─ main.js
└─ pages/
   ├─ star1/
   ├─ star2/
   ├─ star3/
   └─ star4/
```

## 分工建议

- `index.html`：主页面入口，统一跳转到四个子页面。
- `pages/star1/`：第一位同学负责。
- `pages/star2/`：第二位同学负责。
- `pages/star3/`：第三位同学负责。
- `pages/star4/`：第四位同学负责。

每个子页面文件夹内都可以独立维护自己的 `index.html`、`style.css`、`script.js` 和 `images/`，尽量不要修改其他同学的文件夹。

## 子页面统一要求

建议每个子页面至少包含以下内容：

- 返回首页的导航。
- 明星姓名、运动项目和一张主视觉图片。
- 基本信息：国籍、项目、代表身份等。
- 代表成就：冠军、纪录、奖项或关键数据。
- 高光时刻：时间线、图片展示或重要比赛。
- 个人评价：为什么选择这位明星，以及他/她带来的启发。

统一结构可以保证作品整体完整；具体配色、排版、动画和内容风格可以由每位同学自由设计。
