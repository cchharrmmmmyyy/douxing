# 体育明星展示网站前端技术答辩重点版

> 使用场景：这份文档更适合前端设计课程答辩。重点不是介绍运动员是谁，而是说明本项目用了哪些前端知识、这些知识如何在页面中发挥作用、老师深挖时应该怎么回答。

## 一、项目技术定位

本项目是一个纯前端静态展示网站，主要由 HTML、CSS 和 JavaScript 三部分组成。

从前端课程角度看，它不是一个“资料介绍网站”这么简单，而是一个综合练习项目，覆盖了以下知识点：

- HTML 页面结构与语义化标签
- CSS 盒模型、变量、布局、定位、响应式设计
- Flex 布局和 Grid 布局
- CSS 过渡动画、关键帧动画、transform 动画
- JavaScript DOM 操作
- JavaScript 事件监听
- 定时器轮播
- 滚动触发动画
- 选项卡切换
- 图片资源管理
- 可访问性属性
- 移动端适配

项目页面组成：

- `index.html`：主页面，负责整体入口和轮播导航。
- `pages/zy/kobe.html`：科比页面，包含数据动画、时间线动画、名言轮播。
- `pages/star2/index.html`：国乒页面，包含杂志式布局、人物选项卡、滚动动画。
- `pages/star3/index.html`：朱婷页面，偏静态档案式展示，重点体现 HTML 结构和 CSS 排版。
- `pages/star4/index.html`：哈兰德页面，包含复杂首屏动画、数据卡片、相册布局和滚动动画。

答辩开场建议：

“我们这个项目使用的是纯前端技术栈，没有后端和数据库。页面展示主要依靠 HTML 组织结构，CSS 完成视觉设计和响应式布局，JavaScript 负责轮播、点击切换、数字动画和滚动动画。项目的重点是前端页面结构、视觉排版和交互效果的综合应用。”

## 二、HTML 技术点

### 1. HTML 的作用

HTML 负责网页的内容和结构。可以理解为网页的“骨架”。

在项目中，HTML 主要负责：

- 定义导航栏、主视觉区、内容区、卡片区、相册区、底部区域。
- 使用链接实现页面跳转。
- 使用图片标签展示人物图片。
- 使用按钮承载交互操作。
- 使用类名和自定义属性让 CSS、JavaScript 能找到对应元素。

### 2. 语义化结构

项目中大量使用了语义化标签，例如：

- `<header>`：页面头部或导航区域。
- `<nav>`：导航菜单。
- `<main>`：页面主体内容。
- `<section>`：页面中的一个独立内容模块。
- `<article>`：可以独立理解的内容卡片。
- `<footer>`：页面底部。
- `<figure>` 和 `<figcaption>`：图片和图片说明。
- `<dl>`、`<dt>`、`<dd>`：资料列表。

这样写的好处：

- 页面结构更清晰。
- 方便 CSS 分区控制样式。
- 方便搜索引擎和浏览器理解页面内容。
- 方便老师或开发者阅读代码。

老师可能追问：

“为什么不用全是 div？”

建议回答：

“div 没有语义，所有区域都用 div 虽然也能显示，但代码可读性差。我们使用 header、nav、main、section、article 等语义化标签，是为了让页面结构更清楚，也符合前端规范。”

### 3. 页面跳转

主页面通过 `<a>` 标签跳转到四个子页面。

例如：

```html
<a href="./pages/zy/kobe.html">篮球</a>
```

这里的 `href` 是相对路径，表示从当前页面出发，找到对应子页面。

技术解释：

- `./` 表示当前目录。
- `../` 表示上一级目录。
- 子页面返回主页时使用 `../../index.html`，因为子页面在 `pages/star2/` 这类二级目录中，需要回到项目根目录。

老师可能追问：

“为什么子页面返回首页要写 ../../index.html？”

建议回答：

“因为子页面在 pages 文件夹的下一级目录里，比如 pages/star2/index.html，它距离根目录的 index.html 有两层，所以要用 ../../ 回到项目根目录，再访问 index.html。”

### 4. data-* 自定义属性

国乒页面使用了 `data-player`、`data-player-card` 这样的自定义属性。

作用是给元素添加一个机器可读的标识，方便 JavaScript 判断点击的是哪位运动员。

例如：

```html
<button data-player="fzd">樊振东</button>
<article data-player-card="fzd">樊振东介绍</article>
```

技术解释：

- `data-player="fzd"` 表示这个按钮代表樊振东。
- JS 中可以通过 `tab.dataset.player` 读取这个值。
- 点击按钮后，JS 找到对应的卡片并显示。

老师可能追问：

“为什么不用按钮文字判断是谁？”

建议回答：

“按钮文字是给用户看的，可能会改成中文、英文或加样式。data 属性是给程序看的，比较稳定，也更适合做交互逻辑。”

## 三、CSS 基础技术点

### 1. 盒模型 box-sizing

主样式中使用了：

```css
* {
  box-sizing: border-box;
}
```

作用：

让元素的宽度包含内容、内边距和边框，布局时更容易计算。

小白解释：

默认情况下，如果一个盒子设置宽度 300px，再加 padding 和 border，实际宽度会超过 300px。设置 `border-box` 后，整个盒子总宽度仍然是 300px，不容易把布局撑乱。

老师可能追问：

“box-sizing: border-box 有什么好处？”

建议回答：

“它能让元素尺寸更可控，尤其是响应式布局里，避免 padding 和 border 额外撑大元素导致溢出。”

### 2. CSS 变量

主页面和多个子页面使用了 CSS 变量，例如：

```css
:root {
  --bg: #f4f7fb;
  --text: #162033;
  --muted: #657287;
}
```

使用方式：

```css
body {
  color: var(--text);
  background: var(--bg);
}
```

作用：

- 统一管理颜色、字体、间距等设计参数。
- 后期修改主题色更方便。
- 避免重复写很多相同颜色值。

老师可能追问：

“为什么用 CSS 变量，不直接写颜色？”

建议回答：

“直接写颜色也可以，但如果多个地方都用同一个颜色，后期修改很麻烦。CSS 变量可以集中管理设计风格，提高维护性。”

### 3. 字体设置

项目中使用了系统字体和网络字体。

主页面使用：

```css
font-family: Arial, "Microsoft YaHei", sans-serif;
```

部分子页面使用 Google Fonts，引入更有设计感的英文字体。

技术解释：

- `Arial` 是常见英文字体。
- `"Microsoft YaHei"` 是微软雅黑，适合中文显示。
- `sans-serif` 是兜底字体，如果前面的字体没有，就用系统默认无衬线字体。

老师可能追问：

“为什么要写多个字体？”

建议回答：

“这是字体降级方案。如果用户电脑没有第一个字体，浏览器会继续尝试下一个，保证页面文字仍然能正常显示。”

## 四、CSS 布局技术点

### 1. Flex 布局

Flex 主要用于一维布局，也就是一行或一列的排列。

项目中的使用场景：

- 顶部导航栏横向排列。
- 按钮组横向排列。
- 标签组横向排列。
- 卡片内部文字和图片对齐。

常见代码：

```css
.nav-bar {
  display: flex;
  align-items: center;
  justify-content: space-between;
}
```

技术解释：

- `display: flex`：开启弹性布局。
- `align-items: center`：让子元素在交叉轴居中。
- `justify-content: space-between`：让左右元素分散排列，两端对齐。

老师可能追问：

“Flex 适合什么布局？”

建议回答：

“Flex 适合一行或一列的布局，比如导航栏、按钮组、标签组。它可以很方便地控制对齐、间距和换行。”

### 2. Grid 布局

Grid 主要用于二维布局，也就是同时控制行和列。

项目中的使用场景：

- 首页整体区域居中。
- 国乒页面杂志封面布局。
- 朱婷页面双栏资料和成就卡片。
- 哈兰德页面数据卡片、章节卡片、相册布局。

常见代码形式：

```css
.achievement-grid {
  display: grid;
  grid-template-columns: repeat(4, 1fr);
  gap: 16px;
}
```

技术解释：

- `display: grid`：开启网格布局。
- `grid-template-columns`：定义列。
- `repeat(4, 1fr)`：平均分成 4 列。
- `gap`：设置网格之间的间距。

老师可能追问：

“Flex 和 Grid 有什么区别？”

建议回答：

“Flex 更适合一维布局，只关注横向或纵向；Grid 更适合二维布局，可以同时控制行和列。比如导航栏适合 Flex，相册和卡片网格更适合 Grid。”

### 3. 响应式布局

项目中使用了 `@media` 媒体查询适配不同屏幕。

例如：

```css
@media (max-width: 640px) {
  .nav-bar {
    flex-direction: column;
  }
}
```

作用：

当屏幕宽度小于 640px 时，导航栏从横向排列变成纵向排列。

技术解释：

- 电脑屏幕宽，可以多列展示。
- 手机屏幕窄，需要单列展示，防止内容挤在一起。
- 媒体查询可以根据屏幕宽度改变 CSS。

老师可能追问：

“你们项目如何适配手机？”

建议回答：

“我们使用了媒体查询，在不同宽度下调整导航排列、字体大小、卡片布局和轮播高度。比如小屏幕下导航会换行，复杂多列布局会变成单列，保证内容不溢出。”

### 4. clamp 响应式字号

项目中使用了 `clamp()` 控制标题大小。

例如：

```css
font-size: clamp(2rem, 4.4vw, 4rem);
```

技术解释：

`clamp(最小值, 理想值, 最大值)` 表示字体会根据屏幕宽度变化，但不会小于最小值，也不会大于最大值。

小白解释：

它相当于给字体设置了一个“自动伸缩范围”。屏幕大时字体变大，屏幕小时字体变小，但不会无限变大或变小。

老师可能追问：

“为什么不用固定字号？”

建议回答：

“固定字号在不同设备上可能不合适。clamp 可以让标题在大屏上有视觉冲击力，在小屏上又不会撑破布局。”

## 五、CSS 视觉与动画技术点

### 1. position: sticky 固定导航

项目中多个页面使用了固定导航效果：

```css
.site-header {
  position: sticky;
  top: 0;
  z-index: 30;
}
```

作用：

页面滚动时，导航栏会吸附在顶部，方便用户随时切换页面或返回首页。

技术解释：

- `position: sticky`：元素在正常位置时正常滚动，到达指定位置后固定。
- `top: 0`：固定在浏览器顶部。
- `z-index`：控制层级，保证导航栏显示在其他内容上方。

老师可能追问：

“sticky 和 fixed 有什么区别？”

建议回答：

“fixed 是一直固定在屏幕某个位置，脱离文档流；sticky 是先正常占位，滚动到某个位置后再吸附固定，更适合页面导航。”

### 2. backdrop-filter 毛玻璃效果

主页面导航使用了：

```css
backdrop-filter: blur(14px);
```

作用：

让导航栏背景有模糊效果，类似毛玻璃。

技术解释：

它会模糊元素背后的内容，而不是模糊元素本身。通常配合半透明背景一起使用。

老师可能追问：

“backdrop-filter 和 filter 有什么区别？”

建议回答：

“filter 作用在元素本身，比如让图片模糊；backdrop-filter 作用在元素背后的内容，比如半透明导航栏背后的画面被模糊。”

### 3. object-fit 图片裁剪

轮播图和相册中使用了图片填充：

```css
img {
  width: 100%;
  height: 100%;
  object-fit: cover;
}
```

作用：

让图片填满容器，同时保持比例，不被拉伸变形。

技术解释：

- `cover`：图片会覆盖整个容器，超出的部分会被裁剪。
- 适合人物海报、轮播背景、相册封面。

老师可能追问：

“为什么图片没有变形？”

建议回答：

“因为我们使用了 object-fit: cover，它会保持图片原始比例，只裁剪多余部分，不会强行拉伸。”

### 4. transform 动画

项目中轮播、淡入、上移、缩放都大量使用了 `transform`。

主页面轮播核心代码：

```css
.carousel-track {
  transition: transform 680ms ease;
}
```

JavaScript 会改变：

```js
track.style.transform = `translateX(-${current * 100}%)`;
```

技术解释：

- `translateX()` 表示横向移动元素。
- 当前页是第 0 页时，移动 0%。
- 当前页是第 1 页时，移动 -100%。
- 当前页是第 2 页时，移动 -200%。

老师可能追问：

“为什么轮播用 transform，不用 left？”

建议回答：

“transform 动画性能更好，浏览器可以更高效地处理位移和合成，适合轮播、淡入、缩放这类动画。”

### 5. transition 过渡动画

`transition` 用于属性变化时产生平滑过渡。

例如：

```css
.carousel-control {
  transition: background 180ms ease, transform 180ms ease;
}
```

作用：

按钮 hover 时背景和大小变化不会突然跳变，而是平滑变化。

老师可能追问：

“transition 和 animation 有什么区别？”

建议回答：

“transition 通常用于两个状态之间的过渡，比如 hover 前后；animation 更适合复杂连续动画，可以用 keyframes 定义多个阶段。”

### 6. keyframes 关键帧动画

哈兰德页面使用了大量关键帧动画，例如首屏文字、数字、球场线条逐步出现。

关键帧动画的一般形式：

```css
@keyframes heroRise {
  from {
    opacity: 0;
    transform: translateY(22px);
  }
  to {
    opacity: 1;
    transform: translateY(0);
  }
}
```

作用：

定义动画从开始到结束的过程。

老师可能追问：

“什么时候用 keyframes？”

建议回答：

“当动画不是简单 hover，而是页面加载时自动播放，或者需要多个阶段变化时，就适合用 keyframes。”

### 7. prefers-reduced-motion

项目中有：

```css
@media (prefers-reduced-motion: reduce) {
  * {
    transition-duration: 1ms !important;
  }
}
```

作用：

如果用户系统设置了“减少动态效果”，网页会尽量减少动画。

技术意义：

这是一个可访问性设计，照顾对动画敏感或容易眩晕的用户。

老师可能追问：

“这个媒体查询有什么用？”

建议回答：

“这是可访问性优化。如果用户不喜欢动画或系统设置了减少动态效果，页面会自动减少动画时长，提升用户体验。”

## 六、JavaScript 技术点

### 1. DOMContentLoaded

多个 JS 文件使用：

```js
document.addEventListener("DOMContentLoaded", () => {
  // 页面加载完成后执行
});
```

作用：

等 HTML 结构加载完成后再执行 JavaScript，避免 JS 找不到页面元素。

老师可能追问：

“为什么 JS 要等 DOMContentLoaded？”

建议回答：

“因为 JS 需要操作页面元素。如果 HTML 还没加载完就执行，querySelector 可能找不到元素，所以等 DOMContentLoaded 更安全。”

### 2. DOM 查询

项目中常用：

```js
document.getElementById("carouselTrack")
document.querySelectorAll(".carousel-slide")
```

作用：

让 JavaScript 找到页面中的元素，然后修改样式、类名或内容。

区别：

- `getElementById`：根据 id 找单个元素。
- `querySelector`：根据 CSS 选择器找第一个元素。
- `querySelectorAll`：根据 CSS 选择器找一组元素。

老师可能追问：

“querySelectorAll 返回的是什么？”

建议回答：

“它返回的是 NodeList，类似数组但不是完整数组。项目中有时用 Array.from 把它转成数组，方便使用数组方法。”

### 3. 首页轮播原理

首页轮播相关文件：

- `js/main.js`
- `css/main.css`

核心思路：

1. 获取所有轮播项。
2. 用变量 `current` 记录当前显示第几张。
3. 点击下一张时，`current + 1`。
4. 点击上一张时，`current - 1`。
5. 用 `translateX(-current * 100%)` 移动整条轮播轨道。
6. 用定时器每 4 秒自动切换。

核心代码逻辑：

```js
function updateCarousel() {
  track.style.transform = `translateX(-${current * 100}%)`;
}
```

技术解释：

轮播不是每次只移动某一张图片，而是把所有图片放在一个横向轨道中。切换时移动整个轨道，让对应图片进入可视区域。

老师可能追问：

“轮播如何实现循环？”

项目中使用了取模思想：

```js
function normalizeIndex(index) {
  return (index + slides.length) % slides.length;
}
```

建议回答：

“比如一共有 4 张图，当前是第 4 张时再下一张，索引会通过取模回到 0；当前是第 1 张时点上一张，也会回到最后一张。这样轮播就能循环。”

### 4. 动态生成轮播指示点

首页不是手动写死圆点，而是 JS 根据轮播数量生成。

逻辑：

```js
slides.forEach((slide, index) => {
  const dot = document.createElement("button");
  indicators.appendChild(dot);
});
```

作用：

如果以后增加或减少轮播图，指示点数量也会自动变化。

老师可能追问：

“为什么用 JS 创建指示点？”

建议回答：

“这样页面更灵活。轮播图数量变化时，不需要手动改 HTML 里的圆点，JS 会根据实际 slide 数量自动生成。”

### 5. setInterval 定时器

首页轮播和科比名言轮播都使用了 `setInterval`。

```js
timer = setInterval(() => {
  goTo(current + 1);
}, 4000);
```

作用：

每隔 4000 毫秒，也就是 4 秒，自动执行一次切换。

老师可能追问：

“用户手动点击后为什么要 resetTimer？”

建议回答：

“如果用户刚点击完，定时器马上又自动切换，会影响体验。resetTimer 会先清除旧定时器，再重新开始计时，让用户操作后有完整停留时间。”

### 6. addEventListener 事件监听

项目中按钮点击、滚动、触摸都使用事件监听。

例如：

```js
nextButton.addEventListener("click", () => {
  goRight();
  resetTimer();
});
```

作用：

当用户点击下一张按钮时，执行切换函数。

老师可能追问：

“事件监听有什么优势？”

建议回答：

“它可以把 HTML 结构和 JS 行为分开，不需要在 HTML 里写 onclick，代码更清晰，也方便维护。”

### 7. classList 切换类名

项目中经常使用：

```js
element.classList.toggle("is-active", condition);
```

作用：

根据条件添加或移除类名。

在国乒页面中：

- 当前人物按钮添加 `is-active`。
- 当前人物卡片添加 `is-active`。
- 其他人物移除 `is-active`。

技术解释：

JavaScript 不直接写大量样式，而是切换类名；具体显示、隐藏和动画交给 CSS。

老师可能追问：

“为什么 JS 不直接改 display？”

建议回答：

“直接改 display 也能实现，但样式逻辑会分散到 JS 里。用 classList 切换类名，可以让 JS 负责状态，CSS 负责表现，职责更清楚。”

### 8. 国乒人物选项卡原理

相关文件：

- `pages/star2/script.js`

核心逻辑：

```js
const activatePlayer = (player) => {
  tabs.forEach((tab) => {
    const isActive = tab.dataset.player === player;
    tab.classList.toggle("is-active", isActive);
  });

  cards.forEach((card) => {
    card.classList.toggle("is-active", card.dataset.playerCard === player);
  });
};
```

技术解释：

- 点击某个按钮时，读取按钮的 `data-player`。
- 遍历所有按钮，只有匹配的按钮变成激活状态。
- 遍历所有人物卡片，只有对应卡片显示。

老师可能追问：

“为什么使用遍历？”

建议回答：

“因为页面上有多个按钮和多个卡片。遍历可以统一处理所有元素，保证只有当前选择的人物处于激活状态，其他都取消激活。”

### 9. IntersectionObserver 滚动动画

科比、国乒、哈兰德页面都使用了这个技术。

核心代码形式：

```js
const observer = new IntersectionObserver((entries) => {
  entries.forEach((entry) => {
    if (entry.isIntersecting) {
      entry.target.classList.add("is-visible");
      observer.unobserve(entry.target);
    }
  });
});
```

作用：

当某个元素滚动进入屏幕时，给它添加类名，CSS 再播放动画。

技术流程：

1. HTML 元素初始状态是透明、下移。
2. JS 观察这些元素。
3. 元素进入视口后，JS 添加 `is-visible`。
4. CSS 检测到 `is-visible` 后，把透明度变成 1，位置恢复。

老师可能追问：

“为什么不用 scroll 事件一直判断？”

建议回答：

“scroll 事件会频繁触发，如果自己计算每个元素的位置，代码更复杂，也可能影响性能。IntersectionObserver 是浏览器提供的专门 API，更简洁，也更适合滚动曝光动画。”

### 10. requestAnimationFrame 数字动画

科比页面使用 `requestAnimationFrame` 实现数字增长。

核心思想：

```js
function update() {
  current += Math.ceil(target / 60);
  element.textContent = current.toLocaleString();
  requestAnimationFrame(update);
}
```

作用：

让数字从 0 平滑增长到目标值。

技术解释：

`requestAnimationFrame` 会在浏览器下一次重绘前执行回调，适合做动画，比普通循环更流畅。

老师可能追问：

“为什么不用 for 循环？”

建议回答：

“for 循环会瞬间执行完，用户看不到动画过程。requestAnimationFrame 会跟随浏览器刷新节奏逐帧更新，所以视觉上是平滑增长。”

## 八、老师深挖高频问题与回答

### 1. 这个项目为什么说是前端项目？

回答：

“因为项目所有页面展示和交互都在浏览器端完成，没有后端接口和数据库。HTML、CSS、JavaScript 三者共同完成页面结构、样式和交互。”

### 2. HTML、CSS、JavaScript 分别负责什么？

回答：

“HTML 负责内容结构，CSS 负责视觉样式和布局，JavaScript 负责动态交互。比如首页轮播中，HTML 提供轮播图片，CSS 控制它们横向排列和动画过渡，JavaScript 控制当前显示第几张。”

### 3. 页面如何实现响应式？

回答：

“主要通过媒体查询、弹性布局、网格布局和 clamp 字号实现。屏幕变窄时，多列布局会变成单列，导航会换行，图片和字体也会自适应缩放。”

### 4. 首页轮播图的核心原理是什么？

回答：

“所有轮播项横向排列在一个 track 里，每张占 100% 宽度。JS 用 current 记录当前索引，然后设置 track 的 transform: translateX(-current * 100%)，让对应图片移动到可视区域。”

### 5. 轮播如何自动播放？

回答：

“使用 setInterval，每隔 4 秒让 current 加 1，然后更新 transform。当用户点击按钮时，会清除旧定时器并重新计时，避免刚点击就自动跳走。”

### 6. 什么是 DOM？

回答：

“DOM 是浏览器把 HTML 解析成的一棵节点树。JavaScript 可以通过 DOM 找到页面元素，并修改它们的内容、样式或类名。”

### 7. 为什么用 classList，而不是直接写 style？

回答：

“classList 只负责切换状态，具体样式还是写在 CSS 里。这样 JS 和 CSS 分工明确，代码更好维护。”

### 8. IntersectionObserver 的作用是什么？

回答：

“它用来观察元素是否进入浏览器视口。进入后添加类名触发 CSS 动画，适合做滚动出现效果，比手动监听 scroll 再计算位置更简洁。”

### 9. Flex 和 Grid 怎么区分使用？

回答：

“Flex 适合一维排列，比如导航栏、按钮组；Grid 适合二维排列，比如相册、卡片网格、复杂页面分区。”

### 10. transform 和 transition 分别是什么？

回答：

“transform 用来改变元素的位置、旋转、缩放等；transition 用来让这些变化变得平滑。比如轮播里 transform 负责移动，transition 负责移动过程的动画。”

### 11. keyframes 和 transition 有什么区别？

回答：

“transition 是两个状态之间的过渡，比如 hover；keyframes 可以定义完整动画过程，适合页面加载时的入场动画或复杂动画。”

### 12. 为什么使用 object-fit？

回答：

“人物图片比例不一定和容器一致。object-fit: cover 可以让图片填满容器，同时保持原比例，不会被拉伸变形。”

### 13. z-index 的作用是什么？

回答：

“z-index 控制元素层级。比如 sticky 导航栏需要覆盖在页面内容上方，所以设置较高的 z-index。”

### 14. 什么是可访问性？

回答：

“可访问性是让更多用户都能顺利使用网页。项目中使用了 aria-label 辅助说明，也使用 prefers-reduced-motion 照顾不喜欢动画的用户。”

### 15. 如果以后要扩展第五个子页面，哪里需要改？

回答：

“需要新增一个页面文件夹，在主页 HTML 中增加一个轮播 slide 和导航链接。因为指示点是 JS 根据 slide 数量自动生成的，所以圆点不用手动添加太多逻辑。”

## 十、精简版答辩讲稿

老师好，我们项目是一个体育明星主题的纯前端展示网站。虽然内容是体育明星，但我们答辩重点放在前端技术实现上。

项目由一个主页面和四个子页面组成，使用 HTML、CSS 和 JavaScript 完成。HTML 负责页面结构，比如导航、主视觉区、内容区、卡片和相册；CSS 负责样式、布局、动画和响应式；JavaScript 负责轮播、点击切换、数字动画和滚动触发效果。

在 HTML 方面，我们使用了 header、nav、main、section、article、footer 等语义化标签，而不是全部使用 div。这样页面结构更清楚，代码可读性更好。页面跳转使用 a 标签和相对路径完成。

在 CSS 方面，我们使用了 Flex 和 Grid 两种布局方式。Flex 主要用于导航栏、按钮组和标签组，Grid 主要用于卡片网格、相册和复杂页面分区。项目还使用媒体查询做响应式适配，小屏幕下多列布局会自动变成单列，避免内容挤压。

首页轮播是一个核心交互。我们把四张轮播图横向放在一个 track 中，用 JavaScript 变量 current 记录当前索引，然后通过 transform: translateX 改变位置。CSS transition 负责让移动过程平滑。自动播放使用 setInterval，用户点击按钮后会重置定时器，提升交互体验。

科比页面使用 requestAnimationFrame 实现数字递增动画，使用 IntersectionObserver 实现时间线滚动出现，使用 setInterval 和按钮事件实现名言轮播。

国乒页面使用 data-player 自定义属性实现人物选项卡切换。点击按钮后，JavaScript 读取对应人物标识，再通过 classList 切换 is-active 类名，CSS 根据类名控制显示状态。

哈兰德页面重点使用 CSS keyframes 做首屏入场动画，后续内容同样使用 IntersectionObserver 触发滚动出现。图片部分使用 object-fit: cover 保持比例并填满容器，导航使用 position: sticky 保持顶部吸附。

整体来看，这个项目综合使用了前端三大基础技术，重点练习了页面结构、布局设计、响应式适配、动画效果和 DOM 交互，是一个面向前端设计课程的综合展示项目。
