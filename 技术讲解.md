# 一、CSS Grid 网格布局

## 1.1 什么是 Grid？

Grid 翻译过来就是"网格"。你可以想象一个 Excel 表格——有行有列，每个格子可以放东西。**CSS Grid 就是让你在网页上画这样一个表格，然后把内容放进去。**

对比一下：

- **Flexbox**：一维布局，要么横着排、要么竖着排
- **Grid**：二维布局，同时控制行和列

**核心语法就三样：**

1. 用 `display: grid` 把这个容器变成网格
2. 用 `grid-template-columns` 定义有几列，每列多宽
3. 内容自动放进格子里

```css
.container {
  display: grid;
  grid-template-columns: 1fr 1fr 1fr;  /* 三等分 */
  gap: 20px;                            /* 间距 */
}
```

## 1.2 主页：四张明星卡片（四等分）

[css/main.css:129-135](css/main.css#L129-L135)

```css
.hero-carousel {
  width: 100%;
  display: grid;
  grid-template-columns: 1fr;
  align-items: stretch;
}
```

`1fr` 是 Grid 里的一个特殊单位，叫"份"。比如 `1fr 1fr 1fr` 就是三等分。上面代码只有 1 个 `1fr`，意思是"这一列占所有可用空间"。

> **如果要四等分**，只需要写 `grid-template-columns: 1fr 1fr 1fr 1fr`，或者简写 `repeat(4, 1fr)`。

## 1.3 乒乓球页面 Hero 区（两栏布局）

[pages/star2/style.css:92-100](pages/star2/style.css#L92-L100)

```css
.magazine-cover {
  display: grid;
  grid-template-columns: minmax(0, 0.86fr) minmax(420px, 0.94fr);
  gap: clamp(28px, 6vw, 78px);
  align-items: center;
}
```

**逐行解释：**

- `display: grid` → 启用网格
- `grid-template-columns` → 定义两列
  - 第一列：`minmax(0, 0.86fr)` → 最小 0，最大占 0.86 份
  - 第二列：`minmax(420px, 0.94fr)` → **最小 420 像素**，最大占 0.94 份
  - `minmax(最小值, 最大值)` 的意思是：列宽在这个范围内弹性变化
- `gap` → 列之间的间距，用 `clamp()` 实现响应式间距
- `align-items: center` → 两列的内容垂直居中

**效果**：左栏放标题文字，右栏放照片拼贴，两列并排。屏幕宽时按比例分配，屏幕窄到右栏快不够 420px 时，右栏就不继续缩小了。

## 1.5 乒乓球：海报画廊（砖石布局）

[pages/star2/style.css:596-600](pages/star2/style.css#L596-L600)

```css
.gallery-masonry {
  display: grid;
  grid-template-columns: repeat(4, minmax(0, 1fr));
  grid-auto-flow: dense;
  gap: 16px;
}
```

重点在 `grid-auto-flow: dense`。这个词的意思是"紧密排列"。Grid 默认从左到右、从上到下放元素，但如果有些元素特别大（比如跨两列的），dense 模式会自动调整顺序，把后面小的元素"塞进"前面留下的空隙中，形成像砖墙一样的错落布局。

实现大图跨列：

```css
.gallery-card-large {
  grid-column: span 2;   /* 跨 2 列 */
  grid-row: span 2;      /* 跨 2 行 */
}
```

**这是最灵活的 Grid 写法之一。** `auto-fit` 的意思是：自动计算能放几列。

- 如果屏幕宽 300px，只能放 1 列，就变成 1 列

## Grid 常用属性速查

| 属性 | 作用 | 例子 |
|------|------|------|
| `grid-template-columns` | 定义列数和列宽 | `repeat(3, 1fr)` 三等分 |

# 二、响应式布局

## 2.1 弹性容器 `width: min()`

[css/main.css:54](css/main.css#L54)

```css
width: min(1180px, calc(100% - 32px));
```

`min()` 的意思是"取两个值中较小的那个"：

- 当屏幕很宽时，`1180px` 小于 `(100% - 32px)`，所以宽度固定 1180px
- 当屏幕变窄（小于 1212px）时，`calc(100% - 32px)` 更小，所以宽度自动缩小
- 两侧永远留出 32px 的边距（左右各 16px）

**效果**：大屏不超 1180px，小屏自动留边。

---

## 2.2 流畅字号 `clamp()`

[css/main.css:115](css/main.css#L115)

```css
font-size: clamp(2rem, 4.4vw, 4rem);
```

`clamp(最小值, 首选值, 最大值)`：

- 最小 2rem（约 32px）
- 首选 4.4vw（视口宽度的 4.4%）
- 最大 4rem（约 64px）

**效果**：标题字号随窗口宽度平滑变化，不会太小也不会太大。在手机上 2rem，在 1000px 宽的屏幕上约 44px，在大屏上封顶 64px。

---

## 2.3 媒体查询

当上面的弹性伸缩不够时，用媒体查询"断点"来彻底改变布局。

[css/main.css:349](css/main.css#L349)

```css
@media (max-width: 640px) {
  .nav-bar {
    flex-direction: column;   /* 导航栏变成竖排 */
  }
  .hero-copy h1 {
    font-size: clamp(1.8rem, 8.4vw, 2.8rem);  /* 调小标题 */
  }
}
```

各页面的断点：

| 页面 | 断点 |
|------|------|
| 主页 (main.css) | 920px、640px |
| 乒乓球 (star2) | 980px、640px |
| 朱婷 (star3) | 820px |
| 哈兰德 (star4) | 860px、560px |
| 科比 (kobe) | 768px、480px |

**原则**：先用 `clamp()` 和 `min()` 做平滑过渡，再用媒体查询做结构性变化（如多列 → 单列）。

## 2.4 `prefers-reduced-motion` 辅助功能

# 三、JavaScript 动画

## 3.1 轮播图自动切换（主页）

[js/main.js](js/main.js)

### 核心变量

```javascript
let current = 0;    // 当前显示第几张（0 表示第一张）
let timer;           // 计时器
```

### 切换动画

```javascript
function updateCarousel() {
  track.style.transform = `translateX(-${current * 100}%)`;
}
```

`transform: translateX()` 的意思是"沿 X 轴平移"。

- 第 0 张：`translateX(0%)` → 显示第一张
- 第 1 张：`translateX(-100%)` → 向左平移一个轨道的宽度，显示第二张
- 第 2 张：`translateX(-200%)` → 显示第三张

**加上 CSS 过渡**（[css/main.css:153](css/main.css#L153)）：

```css
transition: transform 680ms cubic-bezier(0.16, 0.8, 0.2, 1);
```

这个过渡让平移不再是"跳"过去的，而是 **680 毫秒内平滑滑动**过去。`cubic-bezier` 是一种缓动函数，控制动画"先快后慢"。

### 自动播放

```javascript
function resetTimer() {
  clearInterval(timer);                    // 清除旧的计时器
  timer = setInterval(() => {              // 设置新的计时器
    goTo(current + 1);                     // 每 4000ms(4秒) 切换到下一张
  }, 4000);
}
```

`setInterval` 是浏览器提供的定时函数——每隔 4000 毫秒执行一次里面的代码。

### 用户交互时重置计时器

```javascript
nextButton.addEventListener("click", () => {
  goRight();          // 切换到下一张
  resetTimer();       // 重置计时器，防止刚点完立刻自动切换
});
```

**为什么需要 `resetTimer()`？** 如果不重置，假设还剩 0.5 秒就要自动切换了，你点了一下，切换完 0.5 秒后又自动切换，体验很糟糕。重置后重新倒计时 4 秒。

### 小圆点指示器

```javascript
function renderIndicators() {
  indicators.innerHTML = "";
  slides.forEach((slide, index) => {
    const dot = document.createElement("button");
    dot.classList.toggle("is-active", index === current);
    dot.addEventListener("click", () => {
      goTo(index);
      resetTimer();
    });
    indicators.appendChild(dot);
  });
}
```

遍历每一张幻灯片，创建一个小圆点按钮。当前显示的图片对应的小圆点添加 `is-active` 类（变亮显示）。点击小圆点直接跳到对应图片。

## 3.2 滚动渐入动画（乒乓球 & 哈兰德 & 科比）

[pages/star2/script.js](pages/star2/script.js)

### 什么是 Intersection Observer？

"交叉观察器"。它能监听"一个元素是否进入了视口（viewport，即屏幕可见区域）"。过去人们用 `scroll` 事件监听滚动来检测，但滚动事件每秒触发几十次，很耗性能。Intersection Observer 是浏览器底层实现的，只有元素真的进入/离开视口时才触发。

### 核心代码

```javascript
const observer = new IntersectionObserver(
  (entries) => {
    entries.forEach((entry) => {
      if (entry.isIntersecting) {                  // 如果这个元素进入了视口
        entry.target.classList.add("is-visible");  // 给它加上 "可见" 类
        observer.unobserve(entry.target);          // 停止观察（动画只触发一次）
      }
    });
  },
  { threshold: 0.14 }   // 元素 14% 进入视口时触发
);
```

**完整流程：**

```
页面加载时 → 所有区域默认不可见 + 偏移下方
     ↓
用户滚动页面
     ↓
某个区域的 14% 进入了屏幕
     ↓
Intersection Observer 触发 → 加上 is-visible 类
     ↓
CSS transition 让 opacity 从 0→1、translateY 从 36px→0
     ↓
⚠ 效果：元素从下方升起并逐渐显现
```

### CSS 配合

[pages/star2/style.css:677-688](pages/star2/style.css#L677-L688)

```css
.reveal {
  opacity: 0;                           /* 不可见 */
  transform: translateY(36px);          /* 向下偏移 36px */
  transition: opacity 980ms, transform 980ms;  /* 平滑过渡 */
}

.reveal.is-visible {
  opacity: 1;                           /* 完全可见 */
  transform: translateY(0);             /* 回到原位 */
}
```

### 错开（stagger）效果

```css
.stagger-child {
  transition-delay: calc(var(--stagger-index, 0) * 110ms);
}
```

[JS 部分](pages/star2/script.js#L19-L25) 给每个子元素设置 `--stagger-index` 为 0、1、2、3……

- 第 0 个延迟 0ms
- 第 1 个延迟 110ms
- 第 2 个延迟 220ms

**效果**：四个卡片不是同时出现，而是依次出现，像排队一样。

## 3.3 数字滚动计数（科比页面）

[pages/zy/kobe.js:10-25](pages/zy/kobe.js#L10-L25)

```javascript
function animateCounter(element, target, suffix = '') {
  let current = 0;

  const update = () => {
    current += Math.ceil(target / 60);    // 每帧增加目标值的 1/60
    if (current >= target) {              // 达到目标就停止
      element.textContent = target.toLocaleString() + suffix;
      return;
    }
    element.textContent = current.toLocaleString() + suffix;
    requestAnimationFrame(update);        // 请求下一帧
  };

  update();
}
```

### 什么是 `requestAnimationFrame`？

浏览器的刷新频率通常是每秒 60 次（60fps）。`requestAnimationFrame(update)` 的意思就是："浏览器，你下次刷新屏幕的时候调用 update 函数"。

**执行流程：**

```
调用 update()
  → 显示当前数字
  → 调用 requestAnimationFrame(update)
  → 约 16.7ms 后浏览器刷新
  → 再次调用 update()
  → 数字增加
  → 再请求下一帧……
  → 重复直到达到目标值
```

### 为什么用它而不是 `setInterval`？

| 特性 | `requestAnimationFrame` | `setInterval` |
|------|------------------------|---------------|
| 后台标签页 | **自动暂停**，不浪费性能 | 继续执行，浪费 |
| 帧率 | 自动匹配屏幕刷新率（通常 60fps） | 固定间隔，可能卡顿 |
| 电池 | 省电 | 耗电 |

### 实际执行

用科比的数据举例：

```
科比总分 33643 ÷ 60 ≈ 每帧加 561
33643 ÷ 561 ≈ 60 帧 ≈ 1 秒完成
```

## 3.4 名言轮播（科比页面）

[pages/zy/kobe.js:90-125](pages/zy/kobe.js#L90-L125)

### 循环切换

```javascript
let currentQuote = 0;

function showQuote(index) {
  quotes.forEach((q, i) => {
    q.classList.toggle('active', i === index);  // 只有当前名言加 active 类
  });
}

function nextQuoteFn() {
  currentQuote = (currentQuote + 1) % quotes.length;  // 取余实现循环
  showQuote(currentQuote);
}
```

`(currentQuote + 1) % quotes.length` 是关键：

- 假如有 5 条名言，quotes.length = 5
- 当前是第 4 条（索引 4），(4+1) % 5 = 0 → 回到第一条
- 这就是"循环"的原理

# 四、项目整体技术总结

## 要点速览

```
CSS Grid 布局
├── 等分网格 — repeat(4, 1fr) 四张卡片 / 技术火力
├── 两栏布局 — 0.86fr / 0.94fr Hero区图文并排
├── 砖石布局 — grid-auto-flow: dense 海报画廊
├── 交替布局 — .chapter-card:nth-child(even) 奇偶互换
├── 自适应网格 — auto-fit + minmax 荣誉殿堂自动换列
├── 时间线 — grid-template-columns: repeat(5, ...) 生涯路线
└── 表单布局 — grid-template-columns: 105px 1fr 标签+值

响应式设计
├── 弹性容器 — min(1180px, calc(100% - 32px))
├── 流畅字号 — clamp(2rem, 4.4vw, 4rem)
├── 媒体查询 — 各页面 480px~980px 断点
└── 减少动画 — prefers-reduced-motion 辅助功能

JavaScript 动画
├── 轮播图 — transform: translateX + setInterval 自动播放
├── 滚动渐入 — IntersectionObserver + CSS transition
├── 数字计数 — requestAnimationFrame 60fps 递增
└── 名言轮播 — 取余循环 + fadeIn 动画 + 交互重置计时器
```

## 汇报时可以说的三句话

1. **CSS Grid** 是二维布局工具，用几行代码就能实现等分、交替、砖石、自适应等各种复杂排版，本项目的页面级布局全部基于 Grid 完成。

2. **响应式设计** 采用三层策略：`width: min()` 控制容器弹性，`clamp()` 控制字号平滑缩放，媒体查询在关键宽度做结构性调整。

3. **JavaScript 动画** 全部使用浏览器原生 API（transform、IntersectionObserver、requestAnimationFrame），没有依赖任何第三方库，性能更好、代码更轻量。
